JOURNEY BUDDI — FIGMA MAKE DESIGN PROMPT (V2 — COMPREHENSIVE)
================================================================

Design a complete, production-quality mobile-first web application called "Journey Buddi." Journey Buddi is an AI-powered adaptive travel companion that helps travelers plan extended adventure trips through conversation with an AI character named Buddi, then acts as a live companion during the trip — monitoring weather and environmental conditions, delivering daily briefings, suggesting itinerary adaptations, providing storytelling/audio guides, and giving insider tips/discounts.

The pilot destination is New Zealand — dramatic scenery, volatile "four seasons in one day" weather, and diverse activities from glacier hiking to whale watching to volcanic crossings. All placeholder content should reference New Zealand locations, activities, and real provider names.

The target user is an "Intentional Explorer" aged 28–55 — someone who values experiences over possessions, researches trips extensively but gets overwhelmed, prefers independent travel with guidance, and hates the feeling of missing something or visiting at the wrong time. They currently spend 20–40 hours planning using spreadsheets, Reddit, and TripAdvisor, then follow a rigid plan that doesn't adapt to reality.

================================================================
BRAND IDENTITY & EMOTIONAL TONE
================================================================

Journey Buddi should feel like a beautifully crafted travel journal crossed with a thoughtful productivity app — designed by someone who genuinely loves the outdoors. Think Airbnb's visual warmth meets Notion's clean structure meets the calm confidence of a seasoned hiking guide.

The overall mood is: NATURE-INSPIRED, calm, warm, premium, trustworthy, and adventure-inviting. Opening the app should feel like opening a window to the destination.

It is NOT loud, cluttered, gamified, or corporate. The app respects the traveler's intelligence and their vacation state of mind.

================================================================
DESIGN SYSTEM — FULL SPECIFICATION
================================================================

DESIGN LANGUAGE: NATURE-INSPIRED
---------------------------------

The visual design draws from nature — the mountains, oceans, forests, and skies that travelers come to experience.

Core Principles:
- The interface should feel ORGANIC, INVITING, and CALMING
- Generous whitespace everywhere — the app should breathe, like being outdoors
- Rich destination photography as backgrounds and accents — every screen should evoke adventure
- Smooth, natural animations — elements drift, fade, and breathe rather than snap or flash
- Navigation is intuitive and effortless — the user should never feel lost in the app, only lost in the beauty of their trip planning
- Interactions feel like turning pages of a beautiful travel journal, not switching tabs in a spreadsheet

TYPOGRAPHY
----------

Font Family: Inter throughout the entire app. Use JetBrains Mono only for data values like times, temperatures, distances, condition scores, prices, and booking references.

Type Scale:
- Display: 36px / 2.25rem / Bold 700 — hero text on landing page, trip title
- H1: 30px / 1.875rem / Bold 700 — page-level titles
- H2: 24px / 1.5rem / Semibold 600 — section headers within pages
- H3: 20px / 1.25rem / Semibold 600 — card titles, day headers
- H4: 18px / 1.125rem / Medium 500 — activity titles, subsection headers
- Body: 16px / 1rem / Regular 400 — default text, chat messages, descriptions
- Body Small: 14px / 0.875rem / Regular 400 — secondary info, metadata, captions
- Caption: 12px / 0.75rem / Regular 400 — timestamps, labels, tiny metadata

Line heights: 1.5 for body, 1.3 for headings, 1.6 for chat messages.

SPACING SYSTEM
--------------

Based on a 4px grid:
- xs: 4px — tight spacing within compact components
- sm: 8px — internal component padding, gaps between small elements
- md: 16px — card padding, spacing between related elements
- lg: 24px — spacing between sections within a page
- xl: 32px — major section gaps
- 2xl: 48px — page-level vertical spacing, hero section padding
- 3xl: 64px — hero sections, major visual breaks

BORDER RADIUS
-------------
- Small elements (badges, tags): 20px (full pill shape)
- Buttons: 12px (slightly more organic than sharp rectangles)
- Cards: 16px (softer, warmer corners)
- Chat bubbles: 18px with one corner at 4px
- Input fields: 12px
- Bottom sheets: 20px top corners, 0 bottom
- Condition banners: 12px

SHADOWS
-------
- Card resting: 0 2px 8px rgba(44,62,80,0.06) — softer, warmer than pure black
- Card hovered/pressed: 0 6px 16px rgba(44,62,80,0.1) with translateY(-2px)
- Condition banner: 0 2px 12px rgba(44,62,80,0.08)
- Bottom sheet: 0 -4px 24px rgba(44,62,80,0.12)
- Bottom tab bar: 0 -1px 4px rgba(44,62,80,0.06)
- Floating action button: 0 4px 16px rgba(44,62,80,0.15)

CORE COMPONENT LIBRARY

CARDS
-----
The primary content container. Sand (#ECE5D8) or Cloud (#FAFAFA) background, 16px border-radius, 16px internal padding, subtle warm shadow.

Variants:
- Default Card: Cloud bg, subtle shadow, no border.
- Day Card (Itinerary Dashboard): Cloud bg, contains day number, date, location, activity icons with booking status, accommodation, condition banner, tightness indicator. Tappable.
- Activity Card (Enhanced): Cloud bg, contains activity emoji + name, provider name + star rating + price, time slot, condition badge, booking status badge, action buttons row (Maps / Details / Edit).
- Accommodation Card: Cloud bg, accommodation type icon + name, address, check-in/out times, booking status + reference, cost. Action buttons (Maps / Call / Change).
- Transport Card: Cloud bg, mode icon + origin → destination, distance + duration, scenic stops listed, Google Maps button.
- Story Card: Soft purple tint background, story title + excerpt, story type badge (legend/history/geology), action buttons (Listen / Read).
- Buddi Recommendation Card: Soft teal tint background, Buddi Teal left border (4px), small Buddi avatar, recommendation text, accept/alternative buttons.
- Alert Card: Sunset Amber left border (4px) for warnings. Warm red (#EB5757) left border for urgent.
- Provider Comparison Card: Cloud bg, provider name + location, star rating + review count, price in JetBrains Mono, "What's special" one-liner, booking URL button. Used in rows of 2-3 for comparing providers.
- Hidden Gem Card: Treasure Gold left border, gold tint background, gem name + type badge, proximity to route, "Save to Google Maps" button.
- Tip Card: Tip Blue left border, tip text, category badge (discount/app/logistics), action link if applicable.

CONDITION BANNER
----------------
A prominent banner shown at the top of Day Detail and Daily Briefing pages. Full-width with 12px radius, 16px padding.

Variants based on condition score:
- Excellent/Good (score 70-100): Green-tinted background, forest green text, score number in JetBrains Mono. Message: "Great conditions — perfect day for [activities]"
- Fair (score 40-69): Amber-tinted background, amber text. Message includes: "Mixed conditions" + action button: [🔄 See Alternative Plan]
- Poor/Unsafe (score 0-39): Red-tinted background, red text. Message + two buttons: [🔄 Shuffle My Plan] [💡 Indoor Alternatives]

Always includes:
- Condition emoji + label (GREAT/GOOD/MIXED/POOR)
- Score as "XX/100" in JetBrains Mono
- One-sentence summary of what's driving the score
- Forecast confidence line: "Forecast confidence: High ✓" or "Forecast may change — I'll update as we get closer" or "Too early for reliable forecast"

TIGHTNESS INDICATOR
-------------------
A compact horizontal bar showing how packed a day is.

Visual: A progress bar with color coding:
- 🟢 < 50%: "Relaxed — plenty of breathing room" (green fill)
- 🟡 50-70%: "Comfortable — good balance" (teal fill)
- 🟠 70-85%: "Tight — full day, no room for delays" (amber fill)
- 🔴 > 85%: "Overpacked — consider removing something" (red fill)

Format: "📊 Day Fill: 67% — Comfortable" with the bar below, and a one-line suggestion if applicable.

BOOKING STATUS BADGES
---------------------
Small pill-shaped indicators, similar to condition badges:
- ✅ Booked (green-tinted pill, "Ref: XXX-1234" in Caption)
- ⚠️ Needs Booking (amber-tinted pill)
- ○ Not Booked (gray-tinted pill)
- ❌ Cancelled (red-tinted pill, strikethrough)

BUTTONS
-------
- Primary: Buddi Teal (#2D9CDB) background, white text, 12px border-radius, 14px vertical / 24px horizontal padding, Semibold 600, 16px font. Full-width on mobile for major CTAs.
- Secondary: Transparent bg, Buddi Teal text, 2px Buddi Teal border, 12px radius.
- Ghost: Transparent bg, no border, Stone (#5D6D7E) text.
- Accent: Forest Deep (#1B4332) background, white text — for premium actions.
- Destructive: Warm red (#EB5757) variant. Rarely used.
- Nature Pill: Rounded pill button with organic feel — Sand background, Bark text, used for category filters and tags.

All buttons: 44px minimum height on mobile. Subtle press state.

CHAT BUBBLES
------------
- Buddi's messages: Sand (#ECE5D8) background, border-radius 18px 18px 18px 4px (bottom-left sharp), left-aligned, max-width 85%. Small Buddi avatar (32px circle, Forest Deep background with a friendly compass icon in white) to the left.
- User's messages: Buddi Teal (#2D9CDB) background, white text, border-radius 18px 18px 4px 18px (bottom-right sharp), right-aligned, max-width 75%.

A compact player bar for the storytelling/audio feature. Can be:

================================================================
RESPONSIVE DESIGN
================================================================

Mobile (< 640px): Single column, full-width cards, bottom tab navigation, 16px margins. PRIMARY design target.

Tablet (640–1024px): Optional sidebar, wider cards, more columns for provider comparisons.

Desktop (> 1024px): Sidebar navigation. Itinerary dashboard: cards left + map right. Chat centered at ~600px. Max content width ~1200px.

All breakpoints: 44px minimum touch targets, visible focus states, no hover-dependent interactions.

================================================================
ACCESSIBILITY
================================================================

- All text meets WCAG AA contrast ratios
- Condition/booking status communicated through BOTH color AND text/emoji — never color alone
- Interactive elements have Forest Deep focus outlines
- Screen reader labels on all components
- Outdoor readability: high contrast, avoid light-on-light
- All animations respect prefers-reduced-motion
- Audio stories have text transcript alternatives

================================================================
ANIMATIONS & MICRO-INTERACTIONS
================================================================

Nature-themed animation philosophy: elements drift, fade, and breathe — like clouds passing, water flowing, leaves settling.

- Buddi messages: typing indicator (1-2s) → message drifts in from left
- Choice cards: rise up like morning mist
- Selection: checkmark with a gentle pulse, like a heartbeat
- Condition badges: single warm pulse on first appearance
- Itinerary reveal: map animation tracing the route, then day cards cascade in
- Swap suggestion: before/after cards smoothly trade places when accepted
- Page transitions: smooth slides (left/right for nav, up/down for drill-down)
- Audio player: progress dot glows gently while playing
- Treasure map gems: subtle golden shimmer on first view
- Day cards reorder: smooth drag with drop zone indicators
- Pull-to-refresh: custom nature animation (leaves falling, compass spinning, etc.)

All animations: 200-400ms, ease-out curves. Calm and confident, not bouncy.

================================================================
PHOTOGRAPHY & IMAGERY
================================================================

Stunning New Zealand landscape photography throughout:
- Landing hero: single breathtaking wide landscape
- Day cards: small thumbnail of each day's location
- Activity cards: small square activity/location photo
- Story cards: atmospheric photo related to the story
- Treasure map gems: small evocative photo of each gem
- Empty states: friendly hand-drawn-style illustrations

Photography style: vibrant but natural. Warm tones (golden hour feel). Not over-saturated. Feels like a skilled traveler's best shots.

================================================================
WHAT NOT TO DO — ANTI-PATTERNS
================================================================

- NO static text-dump itinerary. The itinerary is an interactive dashboard.
- NO boring form-based planning. Planning is a conversation.
- NO notification firehose. Calm and intentional.
- NO gamification (streaks, points, badges, leaderboards).
- NO corporate/cold UI. Nature-inspired warmth everywhere.
- NO raw data tables. Always synthesize into human-readable assessments.
- NO alarm-bell urgency for non-safety issues.
- NO hover-only interactions.
- NO tiny or low-contrast text. Outdoor readability is critical.
- NO loose-end swap suggestions. Every change includes a complete action checklist.
- NO generic provider listings. Always compare, differentiate, and recommend.
